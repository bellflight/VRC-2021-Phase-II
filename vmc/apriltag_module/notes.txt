had to modify /etc/nvidia-container-runtime/host-files-for-container.d with two files, one for opencv stuff and one for vpi stuff:

vpi:

dir, /opt/nvidia/vpi1/lib64/
lib, /usr/lib/aarch64-linux-gnu/libEGL.so
lib, /usr/local/cuda-10.2/targets/aarch64-linux/lib/libcudart.so.10.2
lib, /usr/local/cuda-10.2/targets/aarch64-linux/lib/libcufft.so.10
lib, /usr/lib/aarch64-linux-gnu/libopencv_imgcodecs.so.4.1
dir, /opt/nvidia/vpi1/include/vpi/
dir, /opt/nvidia/vpi1/include/vpi/detail/
dir, /opt/nvidia/vpi1/include/vpi/experimental/

opencv:
lib, /usr/lib/aarch64-linux-gnu/libopencv_core.so
lib, /usr/lib/aarch64-linux-gnu/libopencv_core.so.4.1
lib, /usr/lib/aarch64-linux-gnu/libopencv_videoio.so
lib, /usr/lib/aarch64-linux-gnu/libopencv_videoio.so.4.1
lib, /usr/lib/aarch64-linux-gnu/libopencv_imgproc.so.4.1

so that the shared libs are included in the docker container creation from the host

Then ldconfig has to be run in the created container to fix mappings to the shared libs..
